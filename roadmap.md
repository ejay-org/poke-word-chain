## 📝 참고 사항

### 🎯 Phase별 완성 기준

#### Phase 3 완료 시 (Step 17)
✅ **텍스트로 완전히 플레이 가능한 게임이 완성되어야 합니다**
- 포켓몬 이름 입력 → 검증 → 봇 응답 → 끝말잇기 진행
- 두음법칙 정상 작동
- 게임 종료 조건 정상 동작
- 이 시점에서 API 키 없이도 게임 테스트 가능

#### Phase 4 완료 시 (Step 22)
✅ **음성으로도 입력 가능해야 합니다**
- 마이크로 말하면 텍스트로 변환되어 기존 로직 실행
- 텍스트/음성 입력 선택 가능

#### Phase 5 완료 시 (Step 28)
✅ **AI 페르소나가 음성으로 응답합니다**
- 봇이 단순 단어가 아닌 캐릭터 대사 생성
- 대사를 음성으로 출력

### 우선순위 높은 단계
- **Phase 0-3**: API 없이 먼저 게임을 완성하는 것이 핵심
- **Step 8-10**: 게임 로직의 핵심, 철저한 검증 필요
- **Step 17**: 텍스트 기반 게임 완전 동작 확인 (마일스톤)
- **Step 21, 25**: API 연동은 신중하게 테스트

### 반복 작업이 예상되는 단계
- **Step 9**: 한국어 두음법칙 예외 케이스가 많아 지속적 개선 필요
- **Step 26-27**: LLM 프롬프트는 반복적인 튜닝과 테스트 필요
- **Step 36-37**: 성능 최적화는 지속적인 모니터링과 개선 필요

### 기술적 난이도 높은 단계
- **Step 9**: 한국어 언어학적 예외 처리
- **Step 21**: 오디오 스트리밍 및 포맷 변환
- **Step 26**: 효과적인 프롬프트 엔지니어링

### API 비용 절감 팁
- Phase 3까지는 API 키 없이 개발 가능
- STT 테스트는 짧은 단어로만 (비용 절감)
- LLM 프롬프트는 로컬 목업 데이터로 먼저 테스트 후 실제 API 연동
